## 4. 연결 라이브 VS 추출
> - 사용자가 사용하는 DB의 종류에 따라 연결 방식 선택  
~~~
라이브 연결
- 작업에 대해 항상 사용자가 DB로 쿼리를 직접 전송하는 방식
장점 : 신속한 업데이트 가능
단점 : DB 부하 ↑ / 상대적으로 늦은 처리 속도

추출 연결
- DB의 현재 상태를 스캔하여 추출한후 스캔본을 사용자의 로컬 PC에 저장하는 방식
장점 : 빠른 처리 속도 / 모든 종류의 태블로 기능 사용 가능
단점 : 상대적으로 느린 업데이트 주기
~~~
